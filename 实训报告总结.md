# 图书管理系统 —— 实训课程报告总结

---

## 一、选题背景

在信息化时代，传统的手工图书管理方式存在效率低下、易出错、查询不便等问题。随着计算机技术的普及，开发一套简单实用的图书管理系统，能够有效提高图书的借阅、归还、查询和统计效率，降低管理成本。本选题立足于此现实需求，设计并实现了一个基于C++控制台的图书管理系统，旨在将课堂所学的数据结构（链表）、文件I/O、函数模块化等知识综合应用于实际项目开发中。

本系统面向小型图书馆或班级图书角的管理场景，支持图书的增删查改、借阅归还、统计汇总等核心功能，具有界面友好、操作简便、数据持久化等特点。

## 二、方案论证（设计理念）

### 2.1 总体设计思路

系统采用**面向过程**的模块化设计思想，将不同功能划分为独立的头文件（`.h`），每个头文件封装一个界面逻辑。主程序通过**状态机**模式驱动界面切换，各功能模块之间低耦合、高内聚。

选择该设计理念的原因有三：
1. **模块化**：每个功能界面独立编写和维护，便于团队分工与后期扩展；
2. **状态机驱动**：通过枚举类型 `Screen` 管理界面流转，逻辑清晰，不易出现界面跳转混乱；
3. **控制台实现**：使用 Windows API（`windows.h`）实现光标定位、颜色控制和键盘监听，在不依赖图形库的前提下提供良好的交互体验。

## 三、过程论述（设计实现）

### 3.1 核心数据结构设计

系统的数据基础是**单向链表**，选择链表而非数组的原因在于：图书数量不固定，链表支持动态增删，内存利用率更高。

```cpp
struct Book {
    char id[10];          // 图书编号
    char name[50];        // 书名
    char author[30];      // 作者
    int  totalCount;      // 馆藏总量
    int  borrowedCount;   // 已借出数量
    Book* next;           // 指向下一本书的指针
};
```

**图3：Book 结构体定义**

**图前说明**：数据结构是程序的骨架。此图展示了 `Book` 结构体的字段组成，它是整个系统数据存储和流转的基本单元。

**图后说明**：
- `id[10]`、`name[50]`、`author[30]`：使用定长字符数组存储图书的编号、书名和作者信息，适合控制台程序的简单数据管理。
- `totalCount` 和 `borrowedCount`：分别记录馆藏总量和已借出数量，两者之差即为可借数量。通过这两个字段可以判断图书是否可借（`borrowedCount < totalCount`）。
- `next` 指针：指向链表中下一本书的地址，构成了单向链表的数据连接。链表头指针 `head` 在 `main.h` 中定义为全局变量，所有模块共享。

### 3.2 链表操作：节点增删

#### 3.2.1 添加图书（链表尾部插入）

```
head → [Book1] → [Book2] → [Book3] → nullptr
                                    ↑
                               new Book → nullptr  (新节点插入此处)
```

**图4：链表尾部插入示意图**

**图前说明**：此图用于说明图书添加功能的底层实现原理。理解链表的插入操作是掌握系统核心逻辑的关键。

**图后说明**：添加新图书时，系统首先分配新节点内存，然后遍历链表找到尾节点（`next == nullptr`），将尾节点的 `next` 指针指向新节点，完成尾部插入。插入前会遍历链表检查编号是否重复，若重复则提示用户并取消添加。添加成功后调用 `saveBooksToFile()` 将更新后的链表持久化到文件。

代码实现如下（`book_list.h` 第103-151行）：
```cpp
Book* newBook = new Book;
// ... 输入信息 ...
if (head == nullptr) {
    head = newBook;
} else {
    Book* t = head;
    while (t->next != nullptr) t = t->next;
    t->next = newBook;
}
saveBooksToFile();
```

#### 3.2.2 删除图书（链表节点删除）

```
head → [Book1] → [Book2] → [Book3] → nullptr
                 ↑ 删除此节点
head → [Book1] ──────────→ [Book3] → nullptr
        (prev)              (cur->next)
```

**图5：链表节点删除示意图**

**图前说明**：此图展示从链表中删除指定节点的操作过程。与插入操作一样，删除也是链表的基本操作之一。

**图后说明**：删除时使用双指针法（`prev` 和 `cur`），`cur` 遍历链表查找目标节点，`prev` 始终指向 `cur` 的前驱。找到目标后，将 `prev->next` 指向 `cur->next`，使链表跳过被删除节点，然后释放该节点的内存。若删除的是头节点，则直接将 `head` 指向第二个节点。

```cpp
void gotoxy(int x, int y) {
    COORD c; c.X = x; c.Y = y;
    SetConsoleCursorPosition(GetStdHandle(STD_OUTPUT_HANDLE), c);
}

void color(int c) {
    SetConsoleTextAttribute(GetStdHandle(STD_OUTPUT_HANDLE), c);
}
```

### 3.5 借书与还书流程

```
借书流程：
  输入图书编号 → 遍历链表查找 → 未找到? → 提示"未找到"
                              → 全部借出? → 提示"无法借阅"
                              → 可借? → borrowedCount++ → 保存文件 → 提示"借阅成功"

还书流程：
  输入图书编号 → 遍历链表查找 → 未找到? → 提示"未找到"
                              → 未被借出? → 提示"无法归还"
                              → 可还? → borrowedCount-- → 保存文件 → 提示"归还成功"
```

**图8：借书与还书流程图**

**图前说明**：此流程图展示了借书和还书两个核心业务操作的完整逻辑路径。流程图能够清晰表达程序的分支判断和处理步骤，是理解业务逻辑的最佳方式。

**图后说明**：
- **输入阶段**：使用 `_getch()` 逐字符读取键盘输入，支持退格键（`\b`）编辑，支持ESC键（键码27）退出，按回车（`\r`）确认输入。输入过滤只接受字母和数字字符。
- **查找阶段**：从链表头 `head` 开始遍历，使用 `strcmp()` 比对图书编号，找到目标后返回节点指针。
- **业务判断**：
  - 借书时先判断 `borrowedCount >= totalCount`，若图书已全部借出则拒绝操作；
  - 还书时先判断 `borrowedCount <= 0`，若未被借出则拒绝操作。
- **数据更新**：修改 `borrowedCount` 后立即调用 `saveBooksToFile()` 持久化，确保数据一致性。
- **用户反馈**：成功时绿色提示，失败时红色提示，均显示具体的图书名称和剩余数量。

### 3.7 统计功能

```
统计信息的计算方式：

  head → [Book1] → [Book2] → ... → [BookN] → nullptr
           ↓ 遍历累加
  ┌────────────────────────────────────┐
  │ totalBooks      += totalCount      │ → 图书总数（馆藏册数）
  │ totalBorrowed   += borrowedCount   │ → 已借出总数
  │ totalInLibrary  += (totalCount     │ → 在馆总数
  │                   - borrowedCount) │
  └────────────────────────────────────┘
```

**图10：统计功能数据流图**

**图前说明**：此图展示了统计模块如何从链表数据中汇总计算出三个统计指标。数据流图能够清晰展示"输入数据→处理过程→输出结果"的完整链路。

**图后说明**：
- **数据源**：遍历整条链表，每个 `Book` 节点提供 `totalCount` 和 `borrowedCount` 两个原始数据。
- **累加计算**：使用一次链表遍历同时完成三个统计量的累加，时间复杂度为 O(n)。具体而言：
  - `totalBooks` 累加每本书的 `totalCount`，得到馆藏总册数；
  - `totalBorrowedBooks` 累加每本书的 `borrowedCount`，得到已借出总册数；
  - `totalInLibraryBooks` 累加 `totalCount - borrowedCount`，得到在馆可借总册数。
- **结果显示**：用户选择对应的菜单选项后，在独立的界面中显示对应的统计数据。

- **程序运行阶段**：每次借书、还书、添加或删除图书后，立即调用 `saveBooksToFile()` 将整个链表以二进制形式写入 `books.dat`。采用二进制格式的优势在于读写效率高，且保留了结构体的完整内存布局。
- **程序退出阶段**：遍历链表，逐个释放 (`delete`) 每个节点占用的堆内存，防止内存泄漏。
- **文件格式**：二进制格式，使用 `ofstream::write()` 和 `ifstream::read()` 进行整块数据的读写，每次读写一个 `Book` 结构体大小（约92字节）。

4. **界面美观性**：控制台界面风格较为朴素，可考虑使用图形库（如Qt）或网页前端重构界面。

## 五、课程实际（实践）总结

通过本次图书管理系统的实训课程，我在以下方面获得了显著提升：

### 5.1 技术能力收获

1. **C++ 综合编程能力**：从零开始搭建一个完整的应用程序，综合运用了结构体、指针、链表、文件I/O、函数模块化、头文件组织等核心知识。

1. **模块化思维**：学会了如何将一个大程序拆分为多个功能独立的模块，每个模块职责单一，便于开发、测试和维护。

3. **用户交互设计**：学会了从用户角度出发设计界面布局、操作流程和反馈提示，提升了产品的可用性。

本次实训是大学课程学习中的一次重要实践。将一个看似简单的"增删改查"需求落地成一个可运行的系统，过程中经历了需求分析、方案设计、编码实现、测试调试的完整软件开发周期。这不仅巩固了课堂理论知识，更让我认识到：编程不仅是写代码，更重要的是系统性的思考——如何在功能实现与代码质量之间取得平衡，如何让程序既正确又易于维护和扩展。这些经验将为后续更复杂项目的开发打下坚实基础。
